## swithme 고유 협업 규칙

### 1. PC/모바일 셸 분리 — 환경 차이 우선 의심의 구체 적용

swithme는 `#desktop-shell`과 `#mobile-shell`로 셸이 분리되어 있다. 글로벌 "작업 공정 3 — 환경 차이 우선 의심"이 동일 코드 한 환경에서만 실패할 때 적용되며, swithme에서는 다음 차이를 우선 점검한다:

- 이벤트 모델: PC `mousedown/mouseup` vs 모바일 `touchstart/touchend`.
- 모바일 셸이 PC 타이머를 `MutationObserver`로 감시하여 동기화하는 구조 — 한쪽 DOM이 바뀌지 않으면 동기화가 끊긴다.
- UI 변경 시 **양쪽 셸 모두 반영했는지** 변경 직후 점검한다 (CLAUDE.md "항상 적용되는 핵심 규칙" 참조).

### 2. 문서 탐색 원칙 — swithme 디렉토리 가이드

글로벌 "문서 탐색 원칙"을 적용할 때, swithme에서 능동 탐색해야 할 경로는 다음과 같다:

- `directives/` — 표준 작업 절차(SOP), `INDEX.md`를 먼저 읽는다.
- `docs/` — PRD·기획·가이드.
- `활용정보/` — 기능 연구 노트.
- `.claude/rules/doc-routing.md` — 코드 변경 ↔ 문서 매핑 표.

서브에이전트에 전달할 때도 위 경로의 문서 요약을 먼저 참고시킨다.
