# 문서 작성 규칙

> swithme의 모든 마크다운 문서(특히 `docs/`, `활용정보/`, `directives/`)에 적용되는 작성 규칙.

---

## 1. 단일 책임 원칙 (SRP)

각 문서는 **하나의 주제(관심사)** 만 담당한다.

- 하나의 문서에 여러 주제가 혼재되면 탐색 비용이 증가하고 유지보수가 어려워진다.
- 새로운 내용을 추가할 때는 **기존 문서의 주제와 부합하는지 먼저 확인**하고, 주제가 다르면 별도 문서로 분리한다.
- 문서 이동·재구성 시 **기존 위치의 중복 내용을 즉시 제거**할 것. 같은 내용이 두 곳에 존재하지 않도록 한다.

---

## 2. 구현 상태 마커 (✅ / ☐)

`docs/`·`활용정보/`·`directives/`의 **기능 명세 문서**에서 항목별로 다음 두 기호 중 하나를 **제목 옆에 반드시 붙인다**.

| 마커 | 의미 |
|------|------|
| `✅` | 구현 완료 — 코드에 실제 동작하는 기능이 존재 |
| `☐` | 미구현/예정 — 문서에만 있고 코드에는 없음 |

### 목적

코드를 열지 않아도 **철학과 실제 구현의 delta**를 한눈에 파악할 수 있게 한다.

### 운영 원칙

- 기능을 구현·삭제할 때 해당 마커도 **함께 갱신**한다.
- **사실 정보의 단일 출처는 여전히 코드**이며, 마커는 "철학 문서에서 어디까지 진행됐는가"를 가리키는 인덱스 역할만 한다.

---

## 3. 어휘 규칙 — 도메인 중립성

타겟이 학생에 한정되지 않고 **공부·운동·업무·자기개발 등 모든 활동**으로 확장되었으므로, 학습 도메인에 국한된 어휘를 피한다.

| 학습 도메인 어휘 | 도메인 중립 어휘 |
|---|---|
| 과목 (subject) | **카테고리 (category)** |
| 학생 | **유저** |
| 공부 시간 | **활동 시간** |
| 학습 패턴 | **활동 패턴** |
| 학습 데이터 | **활동 데이터** |

위 표를 기준으로 신규 문서를 작성하고, 기존 문서를 수정할 때도 점진적으로 정정한다.
