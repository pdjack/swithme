# 🧠 데이터 분석 및 알고리즘 가이드 (Analysis Logic)

이 가이드는 앱 내 '분석' 기능을 구현하기 위한 데이터 지표 정의, 계산 알고리즘 및 시각화 원칙을 담고 있습니다.

---

## 1. 메타인지 및 패턴 분석 지표

### 실력 진단 알고리즘 (Pre-Analysis)
- **과목별 성적대 분석**: 중하위권(패턴형) vs 상위권(약점집중형)을 구분하여 공부 시간 배분 로직을 생성합니다.
- **약점 가중치**: 유저가 선택한 '피하고 싶은 단원'에 대해 분석 리포트에서 우선순위 경고를 발생시킵니다.

### 복습 주기 알고리즘 (망각 곡선)
- **알림 로직**: 1차(2~3일 뒤), 2차(1주일 뒤), 3차(시험 직전) 주기에 맞춘 자동 알림 시스템.
- **정답률 추적**: 답지를 보고 이해한 문제가 나중에 다시 풀었을 때 정답으로 이어지는지 데이터를 기록합니다.

### '진짜 공부' 판별 지표 (Habit Analysis)
- **뇌 피로도(1-5단계)**: 공부 종료 후 유저가 입력한 피로도와 순공 시간을 매칭하여 '가짜 공부' 확률을 계산합니다.
- **집중력 밀도**: 총 공부 시간 중 스마트폰 사용 시간이나 휴식 시간을 제외한 순수 몰입도를 산출합니다.

---

## 2. 계획 최적화 알고리즘

### 거꾸로 계획법 (Reverse Planning)
- **로직**: 시험 시간표의 역순(마지막 날 과목부터)으로 공부 스케줄을 자동 배치하여, 시험 직전 날에 1일차 과목을 공부하도록 설계합니다.

### 유연성 및 리스크 관리
- **Buffer Day 로직**: 일주일 중 하루(주로 일요일)를 비워두어, 주중 미달성 계획을 이월하여 처리할 수 있도록 실천율을 재계산합니다.
- **실천율 시각화**: 달성도에 따른 색상 변화(예: 파랑-달성, 빨강-미달)를 통해 주말이나 특정 과목에서의 취약 패턴을 분석합니다.

## 3. 앱 기능 구현을 위한 핵심 데이터 포인트

| 분석 항목 | 수집 데이터 | 분석 로직 |
| :--- | :--- | :--- |
| **순공 시간 밀도** | 타이머 작동 시간 vs 휴식 시간 | 총 공부 시간 중 실질적 몰입도 산출 |
| **루틴 일관성** | 기상 시간 및 공부 시작 시각 | 일별 변동성(표준 편차)을 측정하여 루틴화 정도 분석 |
| **약점 집중도** | OX세모 기록 (✕ 표시 개수) | 반복해서 틀리는 키워드 및 단원 우선순위 도출 |
| **집중 환경 상관관계** | 음악 청취 여부 (Yes/No) | 음악 청취 시와 미청취 시의 성취도 점수 비교 리포트 |
| **생체 환경 변수** | 식단(당 수치), 기상 시간 | 식사 시간과 이후 집중력 변화 패턴 분석 |

## 4. 학습자 유형별 맞춤 전략 분석 (Pattern Diagnosis)

데이터 분석 결과에 따라 다음과 같은 학습 전략을 추천합니다.

### 1) 인출 연습 (Retrieval Practice) 필요 유형
- 책 읽기(회독) 시에는 이해되나, 책을 덮으면 내용이 기억나지 않는 유형
- 학습 시간 대비 실전 문제 풀이 정답률이 현저히 낮은 유형
- 단순 반복 읽기에 지루함을 느끼고 자극이 필요한 유형

### 2) 간격 반복 (Spaced Repetition) 필요 유형
- 시험 직후 학습 내용을 모두 잊어버리는 '벼락치기형'
- 진도를 나갈수록 앞부분 내용을 망각하여 진입 장벽을 느끼는 유형
- 스스로 복습 일정을 관리하기 어려워 시스템적 보조가 필요한 유형

### 3) 교차 학습 (Interleaving) 필요 유형
- 개념은 알지만 실전에서 적합한 공식이나 해결책을 찾지 못하는 유형
- 응용력이 부족하고 정형화된 기본 문제에만 강한 유형
- 한 과목 집중 시 집중도가 급격히 떨어져 환기가 필요한 유형

### 4) 메타인지 전략 (Metacognition) 필요 유형
- 성실하게 공부하지만 성적이 정체되어 있는 유형
- 자신의 실수 패턴이나 자주 틀리는 오답 원인을 파악하지 못하는 유형
- 계획은 거창하나 실천 여부 모니터링이 부족한 유형

### 5) 정교화 (Elaboration) 필요 유형
- 무조건적인 암기를 기피하며 논리적 설득력이 중요한 유형
- 단편적인 지식은 많지만 지식 간의 인과관계나 흐름 파악을 힘들어하는 유형
- 배운 내용을 실제 사례나 실무에 즉시 적용하고자 하는 욕구가 큰 유형

---

## 5. UI 시각화 가이드
- **약점 대시보드**: 반복 오답이 있는 단원을 '보양 필요' 카드로 강조.
- **성취도 히트맵**: 달력이 아닌 실천율 기반의 컬러 히트맵으로 학습 패턴 노출.
- **인공지능 타이머**: 유저의 집중도가 떨어지는 시점(지난 데이터 기준)을 학습하여 맞춤형 휴식 권장.
