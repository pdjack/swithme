# UI/UX 일관성 가이드

> **문서 성격**: 여러 파일에 걸치는 **크로스커팅 원칙**만 정리한다.
> 특정 파일에 귀속되는 규칙(네이밍, 이벤트, 상태 동기화, 모달, 아이콘)은 해당 JS 파일 상단 주석에 있다.
>
> | 규칙 | 코드 주석 위치 |
> |------|--------------|
> | 네이밍 컨벤션 (m- 접두어) | `js/mobile.js` 상단 |
> | 이벤트 핸들링 / 모달 규칙 | `js/ui.js` 상단 |
> | 상태 동기화 규칙 | `js/store.js` 상단 |
> | 아이콘 사용 규칙 | `js/icons.js` 상단 |
>
> 사실 정보(ID, 셀렉터, 함수 목록 등)는 추출 스크립트로 확인한다:
> `python3 execution/extract-html-structure.py` | `extract-css-variables.py` | `extract-js-api.py`

## 1. 디자인 원칙

### 1-1. 브랜드 아이덴티티

- **다크 모드 기반**: 모든 화면의 기본 배경은 다크(`--bg`)
- **글래스모피즘**: 카드·패널에 반투명 유리 질감(`--glass`, `--border`) 적용
- **타이포그래피**: 본문 `Inter`, 타이머·제목 `Outfit`
- **새 색상 추가 시**: 반드시 `:root` CSS 변수로 정의
- **브랜드 컬러 우선순위**: 검정(`--bg`) → 파랑(`--primary`) → 하양(`--text-main`) → 노랑(`#FFCC00`, 포인트 그래픽용)

### 1-2. PC ↔ 모바일 일관성 기준

**기본 원칙: 같은 기능은 같은 경험을 제공한다.**

| 항목 | 일치 여부 | 설명 |
|------|-----------|------|
| **기능** | 반드시 일치 | 같은 기능은 양쪽에서 동일하게 동작해야 한다 |
| **시각 스타일** | 반드시 일치 | 같은 컴포넌트의 색상·폰트·여백·둥근 모서리는 동일 |
| **레이아웃** | 달라도 됨 (예외 있음) | PC는 3열 그리드, 모바일은 1~2열 스택 등 화면 크기에 맞게 적응 |
| **인터랙션** | 플랫폼 적응 허용 | PC: 더블클릭/호버, 모바일: 롱프레스/스와이프 (단, 결과는 동일) |

> **판단 기준**: "PC에서 이 작업을 한 사용자가 모바일로 전환했을 때 혼란 없이 같은 작업을 할 수 있는가?"

> **레이아웃 예외 — 할 일 + 타임테이블 동시 노출 화면**
> 대시보드 및 동일한 멘탈 모델을 공유하는 신규 화면(예: 설정 > 습관)은 **PC·모바일 모두 좌(할 일) + 우(타임테이블) 가로 분할 구조**를 유지한다. 모바일이라고 세로 적층/단일 컬럼 탭으로 바꾸지 않는다. 실제 마크업 구조는 `docs/PRD.md` §5-2 (모바일 콘텐츠 영역 다이어그램) 참조.

## 2. 공통 컴포넌트 구현 원칙

**"데이터 로직은 공유, 렌더링은 분리"**

```
좋은 예:
  getGroupedTaskData()     ← 공유 (tasks.js에서 export)
  renderTasks()            ← PC 전용 렌더링
  renderMobileTasks()      ← 모바일 전용 렌더링

나쁜 예:
  renderTasks() 안에서 데이터 가공 + PC HTML 생성을 같이 수행
```

| 규칙 | 설명 |
|------|------|
| 데이터 가공 함수는 한 곳에서 정의 | `getGroupedTaskData()`, `buildSlotMap()` 등 |
| 렌더 함수는 플랫폼별로 분리 | PC/모바일의 HTML 구조가 다르므로 각각 작성 |
| 비즈니스 로직은 공유 | 삭제, 토글, 정렬 등의 로직은 한 곳에서 정의하고 양쪽에서 호출 |

## 3. UI 텍스트 규칙

- 모든 UI 텍스트(섹션 타이틀, 버튼 레이블, 빈 상태 메시지 등)는 **한국어**로 통일
- PC와 모바일에서 같은 섹션의 타이틀은 **동일한 문구** 사용

| 컴포넌트 | 통일 타이틀 |
|---------|-----------|
| 타임테이블 | 타임테이블 |
| 할 일 목록 | 할 일 |
| 타이머 | 타이머 / 스톱워치 |
| 캘린더 | 캘린더 |
| 분석 | 분석 |
| 설정 | 설정 |
| 회고 | 회고 |
