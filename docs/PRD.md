> **문서 성격**: 현행 상태 문서화 (v1.0)
> **최종 수정**: 2026-03-24

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 스윗미 (SwitMe) |
| **한 줄 설명** | 유저가 체계적으로 할 일을 계획하고, 유저의 행동을 바탕으로 분석하여 개선 방안을 모색해주는 앱 |

### 목표 (Goal)

- 유저가 공부, 집안일 등 여러 분야에서의 계획이 분산되어 혼란해하지 않도록 한다.
- 단순히 계획을 하는 것에 그치지 않고, 앱의 분석과 해결 방안을 바탕으로 **목표 이행률 상승**과 같은 성과와 발전을 이룰 수 있도록 한다.

### 비전 (Vision)

현재는 **학생 대상 공부 관련 계획 및 분석**만 가능한 상태이나, 향후 운동 등 **다양한 분야에서의 계획·분석**으로 확장할 예정이다.

## 3. 핵심 기능 정의 (Scope)

### 3-1. MVP 기능

#### 대시보드

| 기능 | 설명 |
|------|------|
| 타이머 / 스톱워치 | 유저가 선택한 과목의 공부 시간을 측정 (카운트다운 또는 스톱워치 모드) |
| 타임테이블 | 측정한 학습 기록을 24시간 × 10분 단위 그리드에 과목 색상으로 표시 |
| 할 일 관리 | 과목별 할 일 추가·삭제·완료 토글 |
| 날짜 설정 | 날짜별로 다른 계획을 세울 수 있도록 날짜 선택 기능 제공 |

#### 분석

| 기능 | 설명 |
|------|------|
| 학습 성향 테스트 | 7문항 테스트를 통해 6종 페르소나 중 하나로 유저 유형 판별 |
| 데이터 분석 | 회고 기록 기반 평균 지표 산출 (1주일 / 1개월 기간 선택) |
| AI 통합 분석 | 테스트 + 데이터 분석 결과를 결합하여 예측 점수 산출 및 처방전 제시 |
| AI 학습 계획 | 최근 학습 데이터 + 페르소나 기반 내일 추천 과제 자동 생성 |

#### 설정

| 기능 | 설명 |
|------|------|
| 과목 관리 | 과목 추가·삭제, 이름 변경, 색상 설정 (대시보드 할 일·타임테이블 표시에 사용) |
| 하루 회고 | 목표 달성률(자동 계산, 20점) + 시간관리·오답정리·복습진행·숙제완수(각 수동 입력, 20점) = 총 100점 |

#### 캘린더

| 기능 | 설명 |
|------|------|
| 월별 캘린더 | 회고 점수와 오늘 날짜 표시, 월 단위 네비게이션 |

### 3-2. 후순위 기능

- 회고 목록 항목을 유저가 직접 수정 가능

## 4. 기술 스택 및 아키텍처

### 기술 스택

| 영역 | 기술 |
|------|------|
| **프론트엔드** | HTML5, Vanilla CSS, Vanilla JavaScript (ES6+ Modules) |
| **디자인 시스템** | 다크 모드, 글래스모피즘(Glassmorphism), Inter + Outfit 폰트 |
| **아이콘** | Lucide Icons (CDN) |
| **차트** | Chart.js (CDN) |
| **상태 관리** | `localStorage` 기반 클라이언트 사이드 저장 |
| **빌드** | Vite |
| **PWA** | `vite-plugin-pwa` (Service Worker 자동 생성, 오프라인 지원) |
| **코드 품질** | ESLint (Flat Config) + Prettier |
| **테스트** | Vitest + jsdom |
| **배포** | Vercel (GitHub 푸시 → 자동 빌드·배포) |

### 아키텍처

```
브라우저 (클라이언트 전용)
├── localStorage ←→ store.js (전역 상태 관리)
│                      ↕
│          ┌───────────┼───────────┐
│          ↓           ↓           ↓
│      timer.js    tasks.js    analysis.js
│      (세션 기록)  (태스크 CRUD) (분석 결과)
│          ↓           ↓
│      timetable.js   ui.js ←→ mobile.js
│      (시각화)       (데스크탑)   (모바일)
│
├── 서버 없음 — 모든 데이터는 브라우저 로컬에 저장
└── PWA로 오프라인 사용 및 앱 설치 지원
```

### 데이터 저장 (localStorage 키)

| 키 | 내용 |
|----|------|
| `switme_tasks` | 할 일 목록 |
| `switme_history` | 학습 세션 기록 |
| `switme_subjects` | 과목 목록 |
| `switme_reflections` | 날짜별 하루 회고 |
| `switme_analysis` | 분석 결과 이력 |

## 5. 화면 구조

PC(데스크탑)와 모바일 두 가지 셸(Shell)로 분리된 레이아웃. 동일한 state를 공유하며, 디바이스에 따라 자동 전환된다.

### 5-1. PC (Desktop Shell)

```
┌─────────┬──────────────────────────────────────┐
│ Side    │ Top Bar (제목 / 날짜 / 몰입모드)       │
│ Nav     ├────────────┬──────────┬───────────────┤
│         │ 할 일 목록  │ 타이머   │ 타임테이블     │
│ (68px)  │ + 하루 회고 │ (히어로) │ (24h 그리드)  │
│         ├────────────┴──────────┴───────────────┤
│ 탭:     │ [캘린더] [분석] [설정] — 탭 전환으로 표시 │
│ 대시보드 │                                        │
│ 캘린더   │                                        │
│ 분석     │                                        │
│ 설정     │                                        │
└─────────┴──────────────────────────────────────┘
```

### 5-2. 모바일 (Mobile Shell)

```
┌──────────────────────┐
│ 헤더 (모드전환/날짜)   │
├──────────────────────┤
│ 타이머 (SVG 링)       │
│ 탭 네비게이션 (4버튼)  │
│ 컨트롤 (RESET/START) │
├──────────────────────┤
│ 콘텐츠 영역           │
│ ┌─────────┬────────┐ │
│ │ 할 일   │타임테이블│ │
│ │ 목록    │        │ │
│ └─────────┴────────┘ │
│ [캘린더/분석/설정]     │
└──────────────────────┘
```

### 5-3. 공유 컴포넌트

| 컴포넌트 | 설명 |
|----------|------|
| Zen Mode Overlay | 전체화면 몰입 모드 (과목명 + 대형 타이머) |
| Task Modal | 할 일 추가 모달 (과목 선택 + 내용 입력) |
