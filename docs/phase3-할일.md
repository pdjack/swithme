## 4. 결제 (IAP — **실결제**, 상품 1개·최저가)

> 근거 PRD §9-6. **선행 조건: `docs/twa-출시-할일.md` D단계(사업자·통신판매업 확인, 판매자 등록, 환불 방침, 연락처).** D-1 답 없이 실결제 착수 금지.

- [ ] ☐ **4-1** 유료로 잠글 기능 **1개** 선정 (PRD §9-2 후보 중)
- [ ] ☐ **4-2** 가격 책정 — **최저가** (규모 완화 방침)
- [ ] ☐ **4-3** Play Console에 인앱 상품 등록
- [ ] ☐ **4-4** TWA에서 Digital Goods API + Payment Request API 연동
- [ ] ☐ **4-5** Firebase로 구매 검증 + 유료 기능 잠금 해제
- [ ] ☐ **4-6** **환불 처리 흐름 구현** — 구매 취소 시 유료 기능 재잠금
- [ ] ☐ **4-7** (유저) **라이선스 테스터로 샌드박스 검증 먼저** — 실결제 전 기술 흐름 확인 (테스트 수단으로는 계속 사용)
- [ ] ☐ **4-8** (유저) **실결제 1회 완주** — 결제 → 입금 → 수수료 차감 확인
- [ ] ☐ **4-9** (유저) **환불 1회 완주** — 요청 접수 → 처리 → 기능 재잠금 확인
- [ ] ☐ **4-10** (유저) 정산 내역·세금 신고 자료 확인

> **샌드박스는 폐기가 아니라 강등이다.** 목표 달성 수단에서 **개발 중 테스트 수단**으로 역할이 바뀌었다(4-7).

- [ ] ☐ **5-1** **개인정보처리방침 갱신** — "수집 없음" 폐기. 계정 정보·활동 데이터·결제 정보 수집 명시

- [ ] ☐ **5-4** `docs/마케팅플랜.md` 포지셔닝이 실제 앱과 일치하는지 재확인
