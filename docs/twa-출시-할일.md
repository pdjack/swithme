## 2. 작업 목록

> **⛓️ 순서 의존 주의:** B-2(앱 생성)에서 앱 서명 지문이 나와야 A-1(도메인 소유 증명)을 만들 수 있다. **유저 작업이 Claude 작업을 막는 구간이 있다.** 아래 §3 순서표를 따를 것.

### A. Claude가 하는 것

| # | 할 일 | 내용 | 선행 조건 | 상태 |
|---|---|---|---|---|
| **A-1** | 도메인 소유 증명 | 안드로이드가 주소창을 감추려면 "이 도메인이 내 것"이라는 증명이 필요하다. 앱 서명 지문을 사이트에 파일로 올린다. **어긋나면 앱이 주소창을 달고 뜬다** | **B-2 완료 필수** (지문이 나와야 함) | ☐ |
| **A-2** | 앱 파일 빌드 | 현 웹을 감싼 안드로이드 앱 파일 생성. 아이콘·이름·색은 기존 것 그대로 사용 | 없음 | ☐ |
| **A-3** | 매니페스트 보강 | 스토어 요구 항목 중 누락분 채움 (앱 고유 ID 등) | 없음 | ☐ |
| **A-4** | 개인정보처리방침 작성 | **스토어 필수. 없으면 등록 자체가 막힌다.** 스윗미는 데이터를 기기에만 저장하고 서버로 보내지 않으므로 내용은 짧다. 단 **실제 접속 가능한 URL이어야** 하므로 사이트에 페이지로 게시 | 없음 | ☐ |
| **A-5** | 아이콘 여백 점검 | 512px 아이콘이 "마스크 적용"으로 설정돼 있다. 안드로이드가 원형·둥근사각형으로 잘라내므로 여백이 부족하면 로고가 잘린다. 확인 후 필요 시 여백 버전 제작 | 없음 | ☐ |

### B. 유저가 하는 것 (Claude 대행 불가)

계정 인증·결제·법적 신고는 본인이 해야 한다.

| # | 할 일 | 내용 | 상태 |
|---|---|---|---|
| **B-1** | 구글 개발자 계정 등록 | **$25 (1회)**. 신분증 인증 필요. **개인 계정은 심사에 며칠~몇 주 소요 — 실질 병목** | ☐ |
| **B-2** | Play Console에서 앱 생성 | 앱 이름·기본 정보 입력. **완료 시 앱 서명 지문이 나오며, 이걸 Claude에게 전달해야 A-1이 진행됨** | ☐ |
| **B-3** | 트랙 설정 | §1에서 정한 트랙으로 설정 | ☐ |
| **B-4** | 데이터 안전 양식 제출 | 수집 데이터 신고. 스윗미는 "수집 없음"이라 간단 | ☐ |
| **B-5** | 스토어 등록 정보 입력 | 설명문, 스크린샷, 카테고리, 콘텐츠 등급 설문 | ☐ |
| **B-6** | 앱 파일 업로드 | A-2 결과물 업로드 | ☐ |

## 3. 실행 순서

```
[§1 트랙 결정]  ← 유저. 여기부터 시작
       ↓
[B-1 개발자 계정 등록] ──── 심사 대기 (며칠~몇 주) ────┐
       ↓                                            │
       │        이 대기 시간 동안 병렬 진행 가능 →  [A-2 빌드]
       │                                          [A-3 매니페스트]
       │                                          [A-4 개인정보처리방침]
       │                                          [A-5 아이콘]
       ↓                                            │
[B-2 앱 생성 → 서명 지문 확보]  ←───────────────────┘
       ↓
[A-1 도메인 소유 증명]   ← 지문 없으면 불가
       ↓
[B-3 트랙] → [B-4 데이터 안전] → [B-5 등록 정보] → [B-6 업로드]
       ↓
[검증 — §4]
```

**요점:** B-1 심사 대기가 가장 길다. 그 시간에 A-2~A-5를 끝내둔다.

## 4. 검증 — 소유자 분리

| 검증 항목 | 소유자 | 자동화 |
|---|---|---|
| 앱 파일이 정상 빌드되는가 | Claude | ✅ 가능 |
| 증명 파일이 사이트에 정상 게시되는가 | Claude | ✅ 가능 |
| **실기기(안드로이드 폰)에서 주소창 없이 뜨는가** | **유저** | ❌ **불가** |
| **타이머·데이터 저장이 앱 내부에서 정상 동작하는가** | **유저** | ❌ **불가** |

> **⚠️ 완료 조건:** 아래 두 항목(유저 몫)을 통과하지 않으면 **"됐다" 판정 불가**다.
>
> **자동 검증만으로는 실패 경로를 잡지 못한다.** 증명 파일이 어긋나면 앱이 주소창을 달고 뜨는데, **이 증상은 실기기에서만 보인다.** 빌드 성공·파일 게시 성공은 이 실패와 무관하게 통과한다.

## 7. 출시 이후 상시 의무

출시로 끝이 아니다. 등록하는 순간 아래가 영구히 따라온다.

| 의무 | 주기 |
|---|---|
| Android 타겟 API 레벨 상향 대응 (재빌드·재제출) | **매년.** 앱을 수정하지 않아도 필수. 미이행 시 검색에서 제외 |
| 개인정보처리방침 최신 유지 | 수집 항목 변경 시 |
| 로그인 도입 시 계정 삭제 경로 제공 (앱 + 웹 양쪽) | 로그인 도입 시 |
| 별점·리뷰 대응 | 공개 트랙 선택 시 상시 |

## 기술 메모

- PWA 매니페스트: `vite.config.js` 내 `VitePWA` 플러그인에 인라인 정의
- 아이콘: `public/pwa-192x192.png`, `public/pwa-512x512.png` (512는 `purpose: 'any maskable'`)
- 매니페스트 `id` 필드 누락 → A-3 대상
- 배포 도메인: `swithme.app` (Vercel)
- 증명 파일 경로: `public/.well-known/assetlinks.json` (A-1 산출물)
- 빌드 도구: Bubblewrap (`@bubblewrap/cli`). JDK + Android SDK 자동 설치. 현재 `java` 미설치 상태
- 개인정보처리방침 게시 경로: 미정 (A-4에서 결정)
