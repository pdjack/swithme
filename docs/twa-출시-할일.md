## 1. ⚠️ 경로 확정 — 비공개 테스트 의무 (선택 아님)

> **이건 결정이 아니다.** 유저 계정이 **개인 계정**이므로 구글 정책상 **공개 출시(프로덕션)를 처음부터 받을 수 없다.** 비공개 테스트로 관문을 통과해야만 공개가 열린다.

### 구글 정책 (2026 기준)

2023년 11월 13일 이후 만든 개인 개발자 계정은 프로덕션 권한을 받기 전에:

1. **비공개 테스트를 돌린다**
2. **실제 테스터 12명 이상**이 참여(opt-in)한다
3. **14일 연속** 유지된다 — 중간에 빠지면 카운트 리셋
4. 그 뒤 **프로덕션 권한을 신청**한다
5. 승인 심사 **최대 7일** (더 걸릴 수 있음)

> 원래 20명이었으나 2024년 12월 11일 12명으로 완화. 14일 기간은 유지.
> 회사(조직) 계정·구 개인 계정은 이 규칙 면제 — 유저는 개인 계정이라 **해당됨**.

### 출시 목표 두 갈래 (유저 판단)

목표(§PRD 10-2 = 학습)에 따라 **어디까지 갈지**는 유저가 정한다:

| 목표 | 어디까지 | 12명×14일 필요? |
|---|---|---|
| **비공개 테스트까지만** | 앱을 테스터에게 배포하고 실기기에서 굴려봄. 스토어 운영 학습 대부분 달성 | ❌ 불필요 (테스터 소수로 충분) |
| **공개 출시까지** | 스토어 검색 노출·공개 별점 | ✅ **필수 관문** |

**☐ 유저 판단 대기:** 비공개까지만으로 학습 목표가 충족되는가, 아니면 공개까지 가는가.
→ 이 답이 **C 단계(관문) 착수 여부**를 가른다.

## 2. 작업 목록

> **⛓️ 순서 의존 주의:** B-1(앱 생성)에서 앱 서명 지문이 나와야 A-1(도메인 소유 증명)을 만들 수 있다. **유저 작업이 Claude 작업을 막는 구간이 있다.** 아래 §3 순서표를 따를 것.

### A. Claude가 하는 것

| # | 할 일 | 내용 | 선행 조건 | 상태 |
|---|---|---|---|---|
| **A-1** | 도메인 소유 증명 | 안드로이드가 주소창을 감추려면 "이 도메인이 내 것"이라는 증명이 필요하다. 앱 서명 지문을 사이트에 파일로 올린다. **어긋나면 앱이 주소창을 달고 뜬다** | **B-1 완료 필수** (지문이 나와야 함) | ☐ |
| **A-2** | 앱 파일 빌드 | 현 웹을 감싼 안드로이드 앱 파일 생성. 아이콘·이름·색은 기존 것 그대로 사용 | 없음 | ☐ |
| **A-3** | 매니페스트 보강 | 스토어 요구 항목 중 누락분 채움 (앱 고유 ID 등) | 없음 | ☐ |
| **A-4** | 개인정보처리방침 작성 | **스토어 필수. 없으면 등록 자체가 막힌다.** 스윗미는 데이터를 기기에만 저장하고 서버로 보내지 않으므로 내용은 짧다. 단 **실제 접속 가능한 URL이어야** 하므로 사이트에 페이지로 게시 | 없음 | ☐ |
| **A-5** | 아이콘 여백 점검 | 512px 아이콘이 "마스크 적용"으로 설정돼 있다. 안드로이드가 원형·둥근사각형으로 잘라내므로 여백이 부족하면 로고가 잘린다. 확인 후 필요 시 여백 버전 제작 | 없음 | ☐ |

### B. 유저가 하는 것 — 앱 등록 (Claude 대행 불가)

계정 인증·법적 신고는 본인이 해야 한다. **개발자 계정은 이미 보유 → 등록 단계 생략.**

| # | 할 일 | 내용 | 상태 |
|---|---|---|---|
| **B-1** | Play Console에서 앱 생성 | 앱 이름·기본 정보 입력. **완료 시 앱 서명 지문이 나오며, 이걸 Claude에게 전달해야 A-1이 진행됨** | ☐ |
| **B-2** | 비공개 테스트 트랙 설정 | 앱 파일을 비공개 트랙에 올림 | ☐ |
| **B-3** | 데이터 안전 양식 제출 | 수집 데이터 신고. 스윗미는 "수집 없음"이라 간단 | ☐ |
| **B-4** | 스토어 등록 정보 입력 | 설명문, 스크린샷, 카테고리, 콘텐츠 등급 설문 | ☐ |
| **B-5** | 앱 파일 업로드 | A-2 결과물 업로드 | ☐ |

### C. 유저가 하는 것 — 공개 출시 관문 (§1에서 "공개까지" 택한 경우에만)

> **비공개 테스트까지만 하기로 하면 이 단계 전체 생략.**

| # | 할 일 | 내용 | 상태 |
|---|---|---|---|
| **C-1** | 테스터 12명 × 14일 | **실제 구글 계정 12개 이상**을 비공개 테스트에 참여시키고 **14일 연속** 유지. 중간 이탈 시 리셋. **혼자 만드는 앱엔 최대 마찰 지점** | ☐ |
| **C-2** | 프로덕션 권한 신청 | 12명×14일 충족 후 Play Console에서 신청. 심사 최대 7일 | ☐ |
| **C-3** | 공개 출시 | 승인 후 프로덕션 트랙으로 전환 | ☐ |

## 3. 실행 순서

```
[§1 목표 갈래 결정: 비공개까지만 vs 공개까지]  ← 유저. 여기부터 시작
       │
       ├─────────── [A-2 빌드]        (계정 있으니 B 대기 없이 즉시 착수 가능)
       │             [A-3 매니페스트]
       │             [A-4 개인정보처리방침]
       │             [A-5 아이콘]
       ↓                    │
[B-1 앱 생성 → 서명 지문 확보] │
       ↓                    │
[A-1 도메인 소유 증명]  ←─────┘   ← 지문 없으면 불가
       ↓
[B-2 비공개 트랙] → [B-3 데이터 안전] → [B-4 등록 정보] → [B-5 업로드]
       ↓
[검증 — §4]   ← 비공개 테스트로 실기기 확인. "비공개까지만"이면 여기서 종료
       ↓
─────── 공개까지 가는 경우에만 ───────
       ↓
[C-1 테스터 12명 × 14일]  ← 최소 14일. 최대 병목
       ↓
[C-2 프로덕션 신청 → 심사 최대 7일]
       ↓
[C-3 공개 출시]
```

**요점:** 계정이 이미 있어 초반 대기가 사라졌다. **새 병목은 C-1(12명×14일)** — 공개까지 갈 때만 발생하며, 사람 12명을 모아야 해서 기술이 아니라 **섭외가 관건**이다.

## 4. 검증 — 소유자 분리

| 검증 항목 | 소유자 | 자동화 |
|---|---|---|
| 앱 파일이 정상 빌드되는가 | Claude | ✅ 가능 |
| 증명 파일이 사이트에 정상 게시되는가 | Claude | ✅ 가능 |
| **실기기(안드로이드 폰)에서 주소창 없이 뜨는가** | **유저** | ❌ **불가** |
| **타이머·데이터 저장이 앱 내부에서 정상 동작하는가** | **유저** | ❌ **불가** |

> **⚠️ 완료 조건:** 아래 두 항목(유저 몫)을 통과하지 않으면 **"됐다" 판정 불가**다.
>
> **자동 검증만으로는 실패 경로를 잡지 못한다.** 증명 파일이 어긋나면 앱이 주소창을 달고 뜨는데, **이 증상은 실기기에서만 보인다.** 빌드 성공·파일 게시 성공은 이 실패와 무관하게 통과한다.
>
> **이 검증은 비공개 테스트 단계에서 수행한다.** 즉 C 단계(공개 관문)를 안 가더라도 실기기 검증은 B 단계에서 이미 끝난다.

## 기술 메모

- PWA 매니페스트: `vite.config.js` 내 `VitePWA` 플러그인에 인라인 정의
- 아이콘: `public/pwa-192x192.png`, `public/pwa-512x512.png` (512는 `purpose: 'any maskable'`)
- 매니페스트 `id` 필드 누락 → A-3 대상
- 배포 도메인: `swithme.app` (Vercel)
- 증명 파일 경로: `public/.well-known/assetlinks.json` (A-1 산출물)
- 빌드 도구: Bubblewrap (`@bubblewrap/cli`). JDK + Android SDK 자동 설치. 현재 `java` 미설치 상태
- 개인정보처리방침 게시 경로: 미정 (A-4에서 결정)
- 12명×14일 정책 출처: Play Console Help — App testing requirements for new personal developer accounts
