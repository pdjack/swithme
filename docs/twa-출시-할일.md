## 1. ⚠️ 경로 확정 — 비공개 테스트 의무 (선택 아님)

> **이건 결정이 아니다.** 유저 계정이 **개인 계정**이므로 구글 정책상 **공개 출시(프로덕션)를 처음부터 받을 수 없다.** 비공개 테스트로 관문을 통과해야만 공개가 열린다.

### 구글 정책 (2026 기준)

2023년 11월 13일 이후 만든 개인 개발자 계정은 프로덕션 권한을 받기 전에:

1. **비공개 테스트를 돌린다**
2. **실제 테스터 12명 이상**이 참여(opt-in)한다
3. **14일 연속** 유지된다 — 중간에 빠지면 카운트 리셋
4. 그 뒤 **프로덕션 권한을 신청**한다
5. 승인 심사 **최대 7일** (더 걸릴 수 있음)

> 원래 20명이었으나 2024년 12월 11일 12명으로 완화. 14일 기간은 유지.
> 회사(조직) 계정·구 개인 계정은 이 규칙 면제 — 유저는 개인 계정이라 **해당됨**.

### ✅ 출시 목표 확정 (2026-07-21) — 공개 출시까지

| 목표 | 어디까지 | 판정 |
|---|---|---|
| 비공개 테스트까지만 | 테스터에게 배포·실기기 확인 | ❌ **기각.** 낯선 유저 반응·공개 별점 관리를 못 겪어 목표 미달 |
| **공개 출시까지** | 스토어 검색 노출·공개 별점·실제 매출 | ✅ **채택** |

→ **C 단계(12명×14일 관문)는 선택이 아니라 필수 경로다.** 근거 PRD §9-6·§10-7.

**참고 — 결제 학습만 놓고 보면 공개는 불필요했다:** 인앱결제 기술 흐름(상품 등록·결제창·구매 검증·기능 해제)은 라이선스 테스터로 **비공개에서도 완주 가능**하다. 공개를 요구하는 것은 결제가 아니라 **실제 정산·세금·유저 반응·공개 별점**이다.

### 비공개 vs 공개 테스트 — 차이 (혼동 방지)

| | 비공개(Closed) | 공개(Open) |
|---|---|---|
| 참여 | 지정한 이메일 목록만 | 누구나(링크·스토어 검색) |
| 스토어 검색 노출 | ❌ | ✅ |
| 공개 별점·리뷰 | ❌ 안 남음 | ✅ **영구 기록(일방향 문)** |
| 12명×14일 관문 충족 | ✅ 여기서만 채움 | ❌ |

## 3. 실행 순서

```
[D-1 사업자·통신판매업 확인]  ← 유저. 병렬로 지금 착수. 답이 늦으면 D-3가 밀림
       │
[A-3 매니페스트]  [A-4 방침]  [A-5 아이콘제작 → A-5′ 등록]  [A-2 빌드]
       ↓                    │
[B-1 앱 생성 → 서명 지문 확보] │
       ↓                    │
[A-1 도메인 소유 증명]  ←─────┘   ← 지문 없으면 불가
       ↓
[B-2 비공개 트랙] → [B-3 데이터 안전] → [B-4 등록 정보] → [B-5 업로드]
       ↓
[검증 — §4]   ← 비공개 테스트로 실기기 확인
       ↓
[C-1 테스터 12명 × 14일]  ← 최소 14일. 최대 병목. 생략 불가
       ↓
[C-2 프로덕션 신청 → 심사 최대 7일]
       ↓
[C-3 공개 출시]
       ↓
[D-2 판매자 등록] [D-3 사업자등록] [D-4 환불 방침] [D-5 연락처]
       ↓
[결제 구현 → docs/phase3-할일.md §4]
```

**요점:** 계정이 이미 있어 초반 대기가 사라졌다. **병목 둘** — ① **C-1(12명×14일)**: 사람 12명 섭외라 기술이 아니라 사람 문제. ② **D-1**: 답을 못 받으면 실결제 착수가 막히므로 **지금 병렬로 시작하는 게 이득**이다.

## 4. 검증 — 소유자 분리

| 검증 항목 | 소유자 | 자동화 |
|---|---|---|
| 앱 파일이 정상 빌드되는가 | Claude | ✅ 가능 |
| 증명 파일이 사이트에 정상 게시되는가 | Claude | ✅ 가능 |
| **실기기(안드로이드 폰)에서 주소창 없이 뜨는가** | **유저** | ❌ **불가** |
| **타이머·데이터 저장이 앱 내부에서 정상 동작하는가** | **유저** | ❌ **불가** |

> **⚠️ 완료 조건:** 아래 두 항목(유저 몫)을 통과하지 않으면 **"됐다" 판정 불가**다.
>
> **자동 검증만으로는 실패 경로를 잡지 못한다.** 증명 파일이 어긋나면 앱이 주소창을 달고 뜨는데, **이 증상은 실기기에서만 보인다.** 빌드 성공·파일 게시 성공은 이 실패와 무관하게 통과한다.
>
> **이 검증은 비공개 테스트 단계에서 수행한다.** 즉 C 단계(공개 관문)를 안 가더라도 실기기 검증은 B 단계에서 이미 끝난다.

## 7. 출시 이후 상시 의무

출시로 끝이 아니다. 등록하는 순간 아래가 영구히 따라온다.

| 의무 | 주기 |
|---|---|
| Android 타겟 API 레벨 상향 대응 (재빌드·재제출) | **매년.** 앱을 수정하지 않아도 필수. 미이행 시 검색에서 제외 |
| 개인정보처리방침 최신 유지 | 수집 항목 변경 시 |
| 데이터 안전 양식 재제출 | 로그인·동기화·결제 도입 시 |
| 로그인 도입 시 계정 삭제 경로 제공 (앱 + 웹 양쪽) | 로그인 도입 시 |
| 별점·리뷰 대응 | 공개 출시(C 단계) 도달 시 상시 |
| **환불 요청 응대** | 실결제 시작 후 상시 |
| **⚠️ 세금 신고 (사업자등록한 경우)** | **부가가치세 1월·7월 / 종합소득세 5월.** 앱을 수정하지 않아도, **매출이 0원이어도** 무실적 신고 필요. 미이행 시 가산세 |

> **⚠️ 오해 방지:** 위 세금 신고는 **앱 변경과 무관하다.** 앱을 고칠 때마다 국가에 신고하는 의무는 존재하지 않는다. 달력에 따라 정기적으로 돌아오는 의무이며, **폐업 신고 전까지 계속된다** — 앱을 내리고 학습을 끝내도 폐업 신고를 안 하면 의무는 남는다.
