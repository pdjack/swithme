# swithme 개선사항 목록

> 유저 제기 문제점·개선 요청 정리본. 우선순위·구현 상태는 별도 판단 필요.

---

## 1. 버그 (동작 오류)

### 타이머 / 스톱워치
- **스톱워치 stop 버그:** stop 누르면 시간이 0이 되거나, 다시 start 눌렀을 때 0이 됨.
- **스톱워치 저장 안 됨:** 재진입 시 로드되긴 하나 바로 타이머 모드로 바뀌면서 시간 초기화됨.
- **일반 재진입 시 타이머 저장 안 됨:** 앱 나갔다 들어오면 타이머 상태 저장 안 됨.
- **모바일 백그라운드 복귀 버그:** 스톱워치로 측정 후 다른 앱 오래 쓰다 재접속하면 타이머 모드로 전환되고 측정 기록 사라짐.

### 기록 / 데이터
- **할 일 수행 기록 소실:** 기록 화면 나갔다가 창 닫은 뒤 재진입하면 기록 사라짐.
- **환경 간 데이터 불일치:** `swithme-delta.vercel.app` 과 `swithme.app` 의 로드·세이브 값이 다름.
- **기록한 시간 즉시 표시 안 됨.**
- **날짜 변경 시 대시보드 반영 지연:** 날짜 바뀌어도 대시보드 설정 날짜가 곧바로 업데이트 안 됨.

### 습관 타임테이블
- **메모 표시 안 됨:** 습관 타임테이블 메모(블럭 메모 포함) 안 보임.
- **수정 패널 불일치:** 습관 타임테이블 수정 시 패널과 대시보드에서 뜨는 패널이 다름.

## 2. UI / UX 개선

### 레이아웃 점검
- **태블릿 가로 모드:** 날짜 셀 겹침.
- **태블릿 웹 플랜 셀 드래그 불가:** 셀 만들 때 다운로드·인쇄·공유 메뉴가 떠서 여러 셀 드래그 선택이 안 됨.

### 습관 타임테이블 표현
- **요일 테이블 색상:** 색 연하게 해서 보이게. 카테고리 흰색 금지.
- **타임테이블 박스 표현법 개선.**

### 기록 블럭
- **기록 블럭 수정** 기능.
- **기록 블럭에 할 일 텍스트 표시.**

### 대시보드
- **할 일 수행 시간 총합 표시.**

### 타임테이블 시작 시간
- **첫 행 시간 06 → 05 변경:** 현재 모든 타임테이블 첫 시간이 06시. 05시부터 시작하도록.

## 3. 기능 추가

- **코드 동기화:** 습관 타임테이블 & 할 일 코드와 대시보드 코드 동기화.
- **날짜 탭 상호작용:** 셀 하나 클릭 시 해당 날짜 할 일 보여주기 + 날짜 이동 버튼. 할 일 보기에서 습관 표시/숨김 토글 가능.
