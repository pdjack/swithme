## 1. 경로 — 플랫폼별로 다르다

### 1-1. Android (Google Play) — 비공개 테스트 의무 (선택 아님)

> **이건 결정이 아니다.** 유저 계정이 **개인 계정**이라 구글 정책상 **공개 출시(프로덕션)를 처음부터 받을 수 없다.**

2023-11-13 이후 만든 개인 개발자 계정은 프로덕션 권한 전에:

1. 비공개 테스트를 돌린다 → 2. **실제 테스터 12명 이상** 참여(opt-in) → 3. **14일 연속** 유지(이탈 시 리셋) → 4. 프로덕션 권한 신청 → 5. 심사 **최대 7일**

> 원래 20명 → 2024-12-11 12명으로 완화. 14일은 유지. 조직 계정·구 개인 계정은 면제 — 유저는 개인 계정이라 **해당됨**.

### 1-2. iOS (App Store) — 테스터 관문 없음, 심사 있음

- **12명×14일 같은 관문 없음.** 유료 개발자 계정($99/년)만 있으면 바로 심사 제출 가능.
- **모든 빌드가 Apple 심사**(통상 1~3일). **TestFlight**로 내부·외부 베타 배포 후 App Store 정식 심사.
- **⚠️ Apple은 "웹뷰 단순 래핑" 앱에 더 엄격**(Review Guideline 4.2). 네이티브 기능 통합(푸시·IAP·저장소)이 거절 방어가 된다.

### 1-3. 공개 출시까지 간다 (2026-07-21 확정, 양 플랫폼)

| 목표 | 판정 |
|---|---|
| 비공개/TestFlight까지만 | ❌ 기각. 낯선 유저 반응·공개 별점을 못 겪어 목표 미달 |
| **양 스토어 공개 출시까지** | ✅ 채택 (근거 PRD §9-6·§10-7) |

**참고 — 결제 학습만 놓고 보면 공개는 불필요했다:** IAP 기술 흐름(상품 등록·결제창·검증·해제)은 샌드박스/테스터로 **비공개에서도 완주 가능**. 공개를 요구하는 것은 결제가 아니라 **실제 정산·세금·유저 반응·공개 별점**이다.

#### D-4에서 정해야 할 것

1. **환불 사유 범위** — 기능 미작동 / 단순 변심 / 실수 구매 중 어디까지
2. **기간** — 구매 후 며칠까지
3. **접수 경로** — D-5 이메일
4. **거절 시 통보 방식**

> **참고 — 자동 환불:** 구글·애플 모두 구매 직후 짧은 기간 요청은 **스토어가 자동 처리**(개발자 개입 없음). 그 이후만 개발자가 판단·응대. **착수 시점:** 결제 구현 단계(`docs/phase3-할일.md` §4)에서 채운다.

#### 규모 완화 방침 (PRD §9-6)

| 항목 | 확정값 |
|---|---|
| 유료 상품 | **1개**, 최저가 (양 스토어 각 최저 티어) |
| 홍보 | 자체 제작 콘텐츠 중심(유튜브·릴스), 유료 광고 최소 또는 0 |

## 3. 실행 순서

```
[A-1 Apple 계정 등록]  [D-1 사업자·통신판매업 확인]   ← 유저. 지금 병렬 착수(둘 다 답 늦으면 뒤가 밀림)
        │
[A-2 Capacitor 구성 → android/ios 프로젝트 생성]      ← Claude
        │
[A-3 메타]  [A-4 방침]  [A-5 아이콘제작 → A-5′ 등록]  [A-6 저장소 점검]
        ↓                         ↓
[B-A1 Android 빌드]        [B-I1 iOS 빌드]
        ↓                         ↓
[B-A2~A5 Play 등록·업로드]  [B-I2~I4 App Store Connect·TestFlight]
        ↓                         ↓
[검증 — §4 실기기]         [검증 — §4 실기기]
        ↓                         ↓
[C-1 테스터 12명×14일]      [C-4 정식 심사 제출]
        ↓                         ↓
[C-2 프로덕션 신청→심사]     [C-5 iOS 공개 출시]
        ↓
[C-3 Android 공개 출시]
        ↓
[D-2 판매자 등록] [D-3 사업자등록] [D-4 환불 방침] [D-5 연락처]
        ↓
[결제 구현 → docs/phase3-할일.md §4 (Google IAP + Apple IAP)]
```

**요점:** 병목 셋 — ① **C-1(Android 12명×14일)**: 사람 문제라 Claude가 못 도움. ② **D-1**: 답 못 받으면 실결제 착수 막힘 → 지금 병렬. ③ **A-1(Apple 계정)**: 본인확인에 수일, iOS 전체를 막으므로 지금 병렬.

## 기술 메모

- **Capacitor 패키지:** `@capacitor/core`, `@capacitor/cli`, `@capacitor/android`, `@capacitor/ios`
- **구성:** `npx cap init` → `capacitor.config.ts`(appId 역DNS·appName·webDir=Vite 빌드 출력 `dist`) → `npx cap add android` / `npx cap add ios` → 웹 빌드 후 `npx cap sync`
- **빌드 산출물:** Android=AAB(Android Studio/Gradle), iOS=archive(Xcode). iOS는 **Mac+Xcode 필수**
- **주소창 증명(assetlinks.json) 불필요** — TWA 전용이었음. 하이브리드 네이티브 셸은 주소창 자체가 없음
- **푸시:** `@capacitor/push-notifications` + FCM(Android)/APNs(iOS) — Phase 3 §3과 연동
- **IAP 플러그인:** `@capacitor-community/in-app-purchases` 또는 RevenueCat — Google Play Billing·Apple StoreKit 추상화 (Phase 3 §4)
- **저장소:** 웹 `localStorage` 동작하나 영속성 위해 `@capacitor/preferences` 검토 (A-6)
- 아이콘: `public/pwa-192x192.png`, `public/pwa-512x512.png` → Capacitor 아이콘 생성 도구로 네이티브 세트 파생
- 배포 도메인: `swithme.app` (Vercel) — 방침·문의 페이지 호스팅용으로 유지
- 정책 출처: Play Console Help — App testing requirements / Apple — App Store Review Guidelines·TestFlight
