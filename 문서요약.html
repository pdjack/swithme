1. 프로젝트 개요

항목내용
프로젝트명스윗미 (SwitMe)
한 줄 설명유저가 체계적으로 할 일을 계획하고, 회고 데이터를 바탕으로 패턴을 분석하여 개선 방안을 제시하는 앱

목표 (Goal)

  • 유저가 여러 분야(공부, 운동, 업무 등)에서의 계획이 분산되어 혼란해하지 않도록 한다.
  • 단순히 계획을 하는 것에 그치지 않고, 회고 데이터 기반 분석과 처방전을 통해 목표 이행률 상승과 같은 성과와 발전을 이룰 수 있도록 한다.

비전 (Vision)

계획과 관리가 필요한 모든 사람이 자신만의 카테고리를 설정하여 계획·실행·분석의 루프를 돌릴 수 있는 범용 생산성 도구.

2. 타겟 사용자

항목내용
타겟계획과 관리가 필요한 모든 사람 (학생, 직장인, 자기개발 등)
핵심 Pain Point카테고리별 계획이 분산되어 관리가 어렵고, 계획만 세우고 실행·분석까지 이어지지 않음

3. 핵심 기능 정의 (Scope)

3-1. MVP — 대시보드

기능설명상태
타이머 / 스톱워치유저가 선택한 카테고리의 활동 시간을 측정 (카운트다운 또는 스톱워치 모드). STOP은 일시정지(값 유지·RESUME으로 재개)이며, 세션 기록은 RESET 또는 타이머 완료 시점에 저장(여러 번 멈췄다 재개해도 한 세션으로 합산). 측정 중·멈춤·선택 모드는 앱 재진입 시 복원.✅ 완료
타이머 시간 설정타이머 숫자 클릭 시 모달에서 시·분·초 셀을 각각 탭하여 텍스트 입력. 프리셋 칩(10·15·30분) 제공. "설정" 버튼으로 값만 적용(시작은 별도 START). 마지막 사용값 자동 기억. 스톱워치 모드 클릭 시 안내 토스트.✅ 완료
타임테이블측정한 활동 기록을 24시간 × 10분 단위 그리드에 카테고리 색상으로 표시✅ 완료
할 일 관리카테고리별 할 일 추가·삭제·완료 토글✅ 완료
날짜 설정날짜별로 다른 계획을 세울 수 있도록 날짜 선택 기능 제공✅ 완료

3-1. MVP — 분석

기능설명상태
종합 점수회고 데이터 기반 기간별(7일/30일) 평균 점수 산출, 이전 기간 대비 증감 표시✅ 완료
카테고리별 트렌드 그래프카테고리별 일별 활동 시간(분)을 멀티라인 차트로 시각화✅ 완료
인사이트트렌드 데이터에서 패턴(요일별 변동, 카테고리 상승/하락, 미활동 카테고리, 회고 기록률)을 자연어 카드로 제시✅ 완료
처방전종합 점수·최약 카테고리·최약 요일을 종합하여 맞춤 개선 방안 제시✅ 완료
기존 학습 성향 테스트(7문항 → 페르소나 판별)는 v2.0에서 제거됨. 유저가 직접 축적한 회고 데이터와 활동 시간을 기반으로 분석한다.

카테고리별 트렌드의 지표: 활동 카테고리(과목 등)별 일별 활동 시간(분). 회고 점수는 항목별로 세분된 카테고리 점수가 없으므로, 활동 카테고리 단위로는 타임테이블 history에서 집계한 활동 시간을 사용한다.

3-1. MVP — 설정

기능설명상태
카테고리 관리카테고리 추가·삭제, 이름 변경, 색상 설정 (대시보드 할 일·타임테이블 표시에 사용)✅ 완료 (UI상 "과목"으로 표시, 코드 리네이밍 예정)
회고 항목 설정사용자가 회고 평가 항목을 직접 추가·삭제·수정❌ 미구현
하루 회고목표 달성률(자동 계산) + 사용자 정의 항목(각 수동 입력) = 총점⚠ 부분 구현 (현재 고정 항목, 사용자 정의 전환 예정)
습관 (위클리 + 매일 반복)요일별·매일 반복할 할 일과 플랜을 등록. 우선순위: 매일 < 요일 < 사용자 수정. 시드 트리거: ① 그날 대시보드 첫 진입(lazy) ② 습관 편집 직후 ③ 새 플랜 탭 생성 시 ④ 기록 탭→플랜 탭 전환 시. 시드 대상은 모든 비-습관 플랜 탭이며 시드 로그는 탭별로 분리되어 재시드를 차단한다.✅ 완료

📖 README 진입점

스윗미 (Sweet Me) — 프리미엄 대시보드. 학생들을 위한 몰입형 학습 관리 대시보드.

주요 기능

  • DashBoard: 목표 달성 및 일일 회고 관리.
  • Deep Focus Timer: 뽀모도로 타이머 및 스톱워치 (Zen 모드 지원).
  • Timetable: 주간/일간 학습 시간표 관리.
  • Subjects: 과목별 색상 지정 및 관리.

🌟 스윗미만의 차별점 (Core Differentiator)

단순한 계획 수립 도구를 넘어, 사용자의 학습 패턴을 데이터로 분석하고 과학적인 개선 방안을 제시하는 지능형 학습 관리 시스템.

  • 실시간 패턴 분석: 공부 시간, 집중도(순공 시간 밀도), 루틴 일관성 데이터로 학습 효율을 실시간 진단.
  • 과학적 솔루션 제언: 망각 곡선(간격 반복), 인출 연습, 교차 학습 등 교육 공학 기반 맞춤형 전략 가이드.
  • 메타인지 강화: 데이터 시각화 리포트로 강점·약점을 객관적으로 파악, 스스로 성장 계획 수립을 도움.

1. 디자인 원칙

1-1. 브랜드 아이덴티티

  • 다크 모드 기반: 모든 화면의 기본 배경은 다크(--bg)
  • 글래스모피즘: 카드·패널에 반투명 유리 질감(--glass, --border) 적용
  • 타이포그래피: 본문 Inter, 타이머·제목 Outfit
  • 새 색상 추가 시: 반드시 :root CSS 변수로 정의
  • 브랜드 컬러 우선순위: 검정(--bg) → 파랑(--primary) → 하양(--text-main) → 노랑(#FFCC00, 포인트 그래픽용)

1-2. PC ↔ 모바일 일관성 기준

기본 원칙: 같은 기능은 같은 경험을 제공한다.

항목일치 여부설명
기능반드시 일치같은 기능은 양쪽에서 동일하게 동작
시각 스타일반드시 일치같은 컴포넌트의 색상·폰트·여백·둥근 모서리는 동일
레이아웃달라도 됨 (예외)PC는 3열 그리드, 모바일은 1~2열 스택 등 적응
인터랙션플랫폼 적응 허용PC: 더블클릭/호버, 모바일: 롱프레스/스와이프 (결과는 동일)
판단 기준: "PC에서 이 작업을 한 사용자가 모바일로 전환했을 때 혼란 없이 같은 작업을 할 수 있는가?"
레이아웃 예외 — 할 일 + 타임테이블 동시 노출 화면: 대시보드 및 동일한 멘탈 모델을 공유하는 신규 화면(예: 설정 > 습관)은 PC·모바일 모두 좌(할 일) + 우(타임테이블) 가로 분할 구조를 유지. 모바일이라고 세로 적층/단일 컬럼 탭으로 바꾸지 않는다. 마크업 구조는 PRD §5-2 참조.
타임테이블 탭바의 기존 인터랙션 (이미 구현됨, 같은 의도의 새 액션은 여기에 추가):
  • 모바일 탭 롱프레스 메뉴: 탭에 대한 모든 메타 액션(이름 변경, 종류 변경, 삭제 등)은 이 메뉴에 항목으로 추가. 새 메뉴를 만들지 않는다.
  • PC 탭 인터랙션: 더블클릭 = 인라인 이름 편집 / 호버 = × 삭제 버튼 / 우클릭 = 모바일 롱프레스 메뉴와 동일 항목·순서.
  • + 버튼: 탭바 우측 끝 단독 아이콘. 탭 종류 선택이 필요하면 같은 메뉴 패턴을 + 버튼 위치에서 띄운다.
  • 계획·기록 알약 토글: 그리드 상단의 흰 알약 토글([기록] [계획]). 같은 패턴을 다른 화면에서도 동일 위치·형태로 재사용.

2. 공통 컴포넌트 구현 원칙

"데이터 로직은 공유, 렌더링은 분리"

좋은 예:
  getGroupedTaskData()     ← 공유 (tasks.js에서 export)
  renderTasks()            ← PC 전용 렌더링
  renderMobileTasks()      ← 모바일 전용 렌더링

나쁜 예:
  renderTasks() 안에서 데이터 가공 + PC HTML 생성을 같이 수행
규칙설명

5. UI 작업 진입 절차

  1. 같은 의도의 기존 UI 컴포넌트가 있는지 grep으로 확인. 메뉴/모달/토글/버튼 등 새 컴포넌트를 제안하기 전 반드시 수행.
  2. 셀렉터·함수명이 grep에 잡히면 그 줄의 본문을 반드시 읽는다. 이름만으로 "안다"고 단정하지 않는다. 컨텍스트 메뉴라면 어떤 항목이, 모달이라면 어떤 입력을 받는지까지 확인.
  3. 새 패턴 제안 전, 기존 패턴(컨텍스트 메뉴/모달/알약 토글 등)에 항목 추가만으로 해결 가능한지 먼저 검토. §1-2 "타임테이블 탭바의 기존 인터랙션" 단락 우선 참조.
위 3단계를 거치기 전 새 UI 컴포넌트(드롭다운/모달/페이지)를 제안하면 즉시 자기 정정한다.

6. 컴포넌트 규칙 — 타이머 설정 모달 (#timer-set-modal)

  • 진입: 타이머 모드에서 PC/모바일 타이머 숫자(#timer-display, #m-timer-display) 클릭. 두 셸 모두 editTimer() 호출.
  • 레이아웃: 3열(시·분·초) 셀 + 콜론 구분 + 프리셋 칩 3개(10/15/30분, 원형) + 하단 알약 버튼(취소/설정).
  • 셀 입력: 셀 = <button>. 탭 시 <input type="number">로 전환, blur 시 클램프(시 0–23, 분/초 0–59) 후 버튼 복귀. 휠 스크롤 없음.
  • 프리셋: 칩 탭 시 셀 값만 갱신, 모달 유지. 자동 시작 X.
  • 확정: "설정" 버튼만 값을 state.timer.seconds·totalDuration에 반영하고 모달 닫힘. 타이머 실행은 별도 START 버튼으로.
  • 취소: 외부 영역 탭 / Esc / "취소" 버튼 → 미반영 닫힘.
  • 초기값: state.timer.totalDuration (없으면 25:00). 모달이 영속화한 switme_timer_last_duration에서 복원.
  • 스톱워치 모드: 타이머 숫자 클릭 시 모달 대신 토스트 "스톱워치는 0부터 시작합니다".
  • 금지: 브라우저 기본 prompt() / alert() 사용 금지.

🚀 PWA·배포 가이드 배포 시

SwitMe는 PWA로 구성되어 브라우저 외에 독립 앱으로 설치하여 사용 가능. vite-plugin-pwa가 Service Worker와 Web App Manifest를 빌드 시 자동 생성.

핵심 설정 (vite.config.js)

항목값설명

skipWaitingtrue새 SW 활성화 시 대기 없이 즉시 교체
clientsClaimtrue새 SW가 즉시 모든 클라이언트 제어

배포 및 업데이트 흐름

GitHub 푸시 → Vercel 자동 빌드·배포 → 앱이 새 SW 능동 확인 → 감지 시 즉시 교체 → 유저 앱 자동 갱신
  • 배포 URL: https://swithme-delta.vercel.app/
  • GitHub에 푸시만 하면 Vercel이 자동 빌드.
  • 앱은 포그라운드 복귀 시(최근앱 재개)와 1시간 주기로 새 SW를 능동 확인한다. 감지 시 skipWaiting + clientsClaim으로 즉시 교체되고 controllerchange가 자동 새로고침한다.
⚠️ autoUpdate만으로는 부족하다. registerType: autoUpdate는 "새 SW가 감지되면 즉시 교체"만 하고, 감지 트리거(새 버전 확인)는 만들지 않는다. 기본 등록 코드는 load 시점에만 확인하는데, 설치형 PWA는 최근앱 재개 시 load가 안 떠 갱신을 놓친다(특히 iOS Safari). 예전엔 이 때문에 유저가 앱을 지우고 캐시를 비워야만 갱신됐다. 능동 registration.update() 호출(포그라운드·주기)이 반드시 있어야 설치형 PWA가 자동 갱신된다.

배포 후 검증 체크리스트

SW·PWA 코드를 손댄 배포는 사람 손으로 1회 확인한다. 브라우저 F5는 항상 load를 띄워 갱신되므로 실패 경로(설치형 재개)를 못 잡는다 — F5만으로 "됐다" 판정 금지.

  • ☐ 설치형 PWA를 폰 최근앱에서 재개(새로고침 X) → 몇 초 내 자동 갱신 확인.
  • ☐ iOS Safari 설치본 있으면 동일 재개 확인(가장 취약한 환경).

🧠 분석 철학 기획 영감

스윗미의 '분석' 기능은 단순한 통계 제공을 넘어, 유저의 고유한 활동 패턴을 진단하고 개선을 위한 명확한 처방전을 제시하는 데 목표가 있다.

1. 핵심 분석 루프

계획 수립 → 실행(타이머) → 회고 기록 → 패턴 분석 → 맞춤 조언
     ↑                                              │
     └──────────────────────────────────────────────┘

유저가 직접 축적한 회고 데이터 기반 분석. 별도 성향 테스트 없이 실제 행동 데이터에서 패턴을 도출.

2. 분석 결과 구조

2-1. 종합 점수 ✅

  • 선택한 기간(7일/30일)의 회고 점수 평균을 산출.
  • ✅ 직전 동일 기간 대비 증감(▲/▼/–)도 함께 표시.

2-2. 카테고리별 트렌드 그래프 ✅

  • 각 카테고리의 활동 시간이 시간 흐름에 따라 어떻게 변화하는지 시각화. (현재 구현은 회고 점수가 아닌 카테고리별 활동 시간(분) 기준)
  • 상승/하락/정체 추세를 한눈에 파악.

2-3. 인사이트 — 트렌드 데이터에서 도출한 패턴을 자연어로 제시.

  • ✅ 요일별 변동 패턴 (예: "화~목 집중도 높고 금요일 급락")
  • ☐ 카테고리 간 상관관계 (예: "운동 점수 높은 날 업무 집중도도 상승")
  • ✅ 상승/하락 추세 감지 (선형 회귀 기울기 ±0.5 임계)
  • ✅ 미활동 카테고리 경고
  • ✅ 회고 기록률 저하 경고

2-4. 처방전 ✅ — 종합 점수·트렌드·인사이트를 종합하여 구체적 행동 방안 제시.

3. 진단 레벨

몰입 에너지 (Focus Density) ☐

  • 지표: 타이머 작동 시간 vs 휴식 간격.
  • 처방: 순수 활동 시간을 계산하여 현재 몰입 중인지, 기력 소진으로 효율 낮은 상태인지 시각화.

루틴 일관성 (Routine Consistency) ☐

  • 지표: 활동 시작 시각의 표준 편차 + 주간 실천율 변동성.
  • 처방: 불규칙한 생활 패턴이 효율 저하 원인인지 진단.

4. 패턴별 자동 리포트 시나리오

상태발견된 패턴분석 진단스윗미의 추천 (Solution)
☐시간은 많으나 성과 저조비효율적 활동 확률 높음활동 방식 점검 및 집중 세션 유도
☐특정 요일/시간대 하락환경적 집중 저해요인 발생해당 시간대에 카테고리 전환으로 주의 환기 제안
☐계획 미달성 지속무리한 목표 설정주간 계획 15% 감축 및 Buffer Day 확보 권장
☐특정 카테고리만 지속 하락해당 영역 동기 저하목표 재설정 또는 접근 방식 변경 제안

5. 차세대 기능 가이드

  • ☐ 맞춤형 모드 추천: 유저 집중 패턴에 따라 '몰입 모드(Zen)'와 '회고 모드'를 지능적으로 실시간 추천.
  • ☐ 환경 상관분석: 외부 데이터(음악, 식단 등)와 활동 성과의 상관관계를 분석하여 유저만의 '최적 환경' 리포트 발행.

💡 스마트 가이드 (Smart Guidance) 기획 영감

데이터 분석 결과에 따라 사용자가 즉각 실행할 수 있는 생산성 전략을 제공한다.

1. 패턴 데이터 기반 맞춤형 전략

몰입 밀도 저하 시: 집중 세션 전환

  • 분석 결과: 활동 시간 대비 성취감이 낮거나 멍하니 있는 시간이 많을 때.
  • 실행: 현재 카테고리의 활동 방식을 점검하고, 짧고 집중적인 세션(25~30분)으로 전환.

정체기 극복: 카테고리 교차 (Interleaving)

  • 분석 결과: 한 카테고리만 오래 붙잡고 있어 집중 효율이 급격히 떨어지는 시점.
  • 실행: 성격이 다른 2~3개 카테고리를 섞어서 진행. 뇌의 유연성을 높여 효율 극대화.

2. 대시보드 데이터 활용 루틴

목표 달성률 최적화 — 계획 실천율이 70% 미만이면 마감일로부터 역순으로 분량을 재조정. 무리한 계획보다 완수 가능한 '작은 성공'의 반복이 중요.

시간 관리의 함정 — 타이머 시간만 채우는 비효율적 활동 주의. 회고에서 집중도·몰입도를 함께 기록하여 가장 효율적인 시간대를 찾는다.

3. 생산성 최적화 상식

  • 메타인지 강화: 잘하는 것과 부족한 것을 명확히 구분하는 것이 개선의 시작. 회고 점수가 지속적으로 낮은 카테고리를 우선 점검.
  • 환경 변수 통제: 특정 환경이 집중도를 낮추는 패턴이 확인되면 환경을 조정한 후 집중 모드를 활용.
